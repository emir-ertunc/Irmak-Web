## Görsel Düzen İlhamları

- [x] Ürün görselleri büyük, net ve ürünün el yapımı detayını gösterecek yakınlıkta kullanılmalıdır.
- [x] Pembe, çiçekli ve neşeli atmosfer ürünlerin önüne geçmeden arka plan, rozet, ikon ve geçişlerde hissedilmelidir.
- [x] Ana sayfada ilk ekran güçlü olmalı; ama ürün koleksiyonlarından bir parça ilk ekranda görünerek alışverişe geçiş hissi vermelidir.
- [x] Ürün grid yapısı mobilde iki sütuna kadar düşebilmeli, masaüstünde rahat taranan dengeli kartlar kullanmalıdır.
- [x] El yapımı ürünlerde hafif kusur, üretim farkı, tek adetlik stok ve bakım bilgisi şeffaf şekilde gösterilmelidir.
- [x] Özel sipariş ürünleri normal ürün gibi satılmamalı; önce talep, referans görsel ve ön onay mantığıyla ilerlemelidir.

## Ürün Kartı İçin İyi Örnekler

- [x] Kapak görseli kartın ana odağı olmalıdır.
- [x] Ürün adı, fiyat, stok durumu ve kısa kategori etiketi kart içinde hızlı okunmalıdır.
- [x] Renk veya model varyantı varsa küçük swatch veya mini seçim göstergesi kullanılmalıdır.
- [x] Favori butonu her kartta sabit ve kolay erişilir olmalıdır.
- [x] Hover veya dokunma durumunda kart hafif yükselmeli, çiçek veya parıltı mikro hareketi kısa süreli görünmelidir.
- [x] Hızlı sepete ekleme sadece varyantsız ürünlerde doğrudan çalışmalı; varyantlı ürünlerde ürün detayına yönlendirmelidir.
- [x] Tükendi, tek kaldı, özel sipariş ve yeni ürün rozetleri net ama kalabalık yaratmayacak şekilde kullanılmalıdır.

## Ürün Detayı İçin İyi Örnekler

- [x] Görsel galeri ürün detayını ve ölçeğini gösterecek çoklu görsellerle kurulmalıdır.
- [x] Fiyat, stok, üretim süresi ve sepete ekleme alanı ilk bakışta bulunmalıdır.
- [x] Malzeme, bakım, ölçü, aparat tipi ve alerji uyarısı ayrı okunabilir bloklarda gösterilmelidir.
- [x] Varyant seçimleri açılır menüye saklanmadan buton, swatch veya segmented kontrol olarak gösterilmelidir.
- [x] Kişiselleştirme notu ürün detayında ürünle birlikte bağlam içinde alınmalıdır.
- [x] Özel sipariş ürünlerinde referans görsel, istek notu ve onay süreci açık anlatılmalıdır.
- [x] Benzer ürünler ve aynı koleksiyondan ürünler detay sayfasının sonunda sunulmalıdır.

## Sepet ve Sipariş Talebi İçin İyi Örnekler

- [x] İlk sürüm sipariş talebi modeliyle ilerleyeceği için ödeme hissi yerine "talep oluşturma" dili kullanılmalıdır.
- [x] Sepette her ürün için seçilen varyant, kişiselleştirme notu, üretim süresi ve adet görünmelidir.
- [x] Sepet satırında adet değiştirme, kaldırma ve ürüne geri dönme işlemleri kolay olmalıdır.
- [x] Teslimat ve iletişim formu kısa tutulmalı, zorunlu alanlar açık işaretlenmelidir.
- [x] Kullanıcı hesabı zorunlu tutulmamalıdır.
- [x] Sipariş talebi oluşturulduktan sonra admin panelinde "Yeni Talep" durumuna düşmelidir.
- [x] Başarı ekranında talebin alındığı, sahibinin iletişime geçeceği ve ödeme/kargo bilgisinin sonra netleşeceği yazmalıdır.
